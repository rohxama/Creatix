# Creatix Cafe Design System

## Overview

The Creatix Cafe Design System is a comprehensive, warm, elegant, and welcoming design system for our premium coffee shop mobile application. It follows mobile-first design principles inspired by apps like Starbucks, Blue Bottle Coffee, Blank Street Coffee, and Notion's clean UI.

## Design Principles

### 1. Warmth
- Use warm brown and cream color palette
- Apply rounded corners to all components
- Use soft shadows for depth
- Create cozy, inviting layouts

### 2. Elegance
- Use generous white space
- Keep layouts simple and uncluttered
- Use consistent spacing and typography
- Avoid unnecessary decorative elements

### 3. Simplicity
- Use clear visual hierarchy
- Minimize cognitive load
- Provide clear call-to-actions
- Use familiar patterns

### 4. Consistency
- Use design tokens for all values
- Follow the component library
- Maintain consistent spacing
- Use consistent typography scale

### 5. Accessibility
- Support dynamic font sizes
- Ensure sufficient color contrast
- Provide accessibility labels
- Support screen readers

## Usage

### Importing Design Tokens

```typescript
import { colors, typography, spacing, borderRadius, shadows } from "@/theme";
```

### Using in Components

```tsx
import { View, Text } from "react-native";
import { colors } from "@/theme";

export const MyComponent = () => (
  <View style={{ backgroundColor: colors.light.primary }}>
    <Text style={{ color: colors.light.primaryForeground }}>
      Hello World
    </Text>
  </View>
);
```

### Using NativeWind

```tsx
import { View, Text } from "react-native";

export const MyComponent = () => (
  <View className="bg-primary p-4">
    <Text className="text-primary-foreground">Hello World</Text>
  </View>
);
```

## Maintenance

### Adding New Colors
1. Add to `palette.ts`
2. Add to `light.ts` and `dark.ts`
3. Update `tailwind.config.js`
4. Update `global.css`

### Adding New Typography
1. Add to `typography/index.ts`
2. Update `tailwind.config.js`

### Adding New Components
1. Create component in `src/components/design-system/`
2. Add design guidelines to `guidelines.ts`
3. Export from `index.ts`

---

## Best Practices

1. **Always use design tokens** - Never hardcode values
2. **Follow the component library** - Reuse existing components
3. **Test on multiple devices** - Ensure responsive design
4. **Support both themes** - Light and dark mode
5. **Maintain accessibility** - Follow WCAG guidelines
6. **Document changes** - Update this file when making changes
